Testing Paragraphs:

This is a paragraph containing only one sentence.

This is also a paragraph. Notice the spacing placed between these two lines of text? That's because the paragraph tag is known as a "block-level" tag.

Block-level tags take up ALL the space available to them on the web browser/ screen the web page is being viewed on. They also usually have some "padding" added around them.

In HTML, "padding" refers to empty space surrounding the element and/or tags being used.


Testing In-Line Tags/ Elements:

There are also "in-line" tags and elements. These take up ONLY the space absolutely necessary to display the content.

Usually, these are links, images, and styling/ formatting tags.

Two of the most common in-line tags are em and strong.

Testing Lists:

There are three types of lists in HTML: unordered (bullet lists), ordered (numbered lists), and description lists.

To create an unordered list, you need to use the "ul" element. For each bullet point, you use the "li" (list item) element.

Testing Unordered Lists:

  • This
  • is
  • an
  • unordered
  • list.

Ordered (numbered) lists are created the exact same way. The only difference is the element name. Instead of "ul", we'll use "ol".

Testing Ordered Lists:

  1. This
  2. is
  3. an
  4. ordered
  5. list.

Description lists are where things get a little more complicated.

To create a description list, you need the "dl" element (description list). Nested inside that, we need a "dt" element (description term). Nested inside THAT, we need a "dd" element (description definition).
